# ProposalBase 設計書

---

## Phase 1：要件定義 ✅

### 目的・背景

自治体への委託・指定管理者プロポーザルに参加する法人が、提案書作成を効率化するためのWebアプリ。  
自社の強み・実績をDB化し、案件ごとに再利用することで、毎回ゼロから書く手間をなくす。

現状の課題：
- 自社情報（強み・実績）が散在していて毎回探す手間がかかる
- 案件ごとに提案書をゼロから作っていて非効率
- 自治体ごとの案件情報・進捗がバラバラに管理されている
- 提案書の構成は自治体が公開する雛形に従う必要があり、毎回確認が必要

### ターゲットユーザー

自治体へのプロポーザルを繰り返し行う法人・企業の担当者

具体的には：
- NPO・社会福祉法人・民間企業など
- 年に複数回プロポーザルに参加する組織
- 提案書作成を担当する営業・企画職

### 解決する課題

| 課題 | 解決方法 |
|------|---------|
| 自社情報が散在している | 自社ナレッジDBに一元管理 |
| 毎回ゼロから提案書を書く | 過去文章を引用して再利用 |
| 案件の進捗管理が煩雑 | 案件管理機能で一元管理 |
| 締切を見落とす | ダッシュボードで締切を可視化 |

### v1 主要機能一覧

| # | 機能 | 概要 |
|---|------|------|
| 1 | 案件管理 | 自治体名・案件名・締切・ステータス・募集要項URL・雛形ファイル添付 |
| 2 | 提案書作成 | 自治体の雛形に沿ったセクション登録・本文入力・自社DBからの引用 |
| 3 | 自社ナレッジDB | 強み・実績・会社概要の登録・タグ検索 |
| 4 | 過去提案書閲覧 | 案件ごとの提案書履歴・検索 |
| 5 | ダッシュボード | 進行中案件一覧・締切カレンダー |

### 提案書作成の考え方

- 提案書のセクション構成は**自治体が公開する雛形で決まる**（ユーザーが自由に決めるものではない）
- ユーザーは雛形を参照しながらセクション名を手動登録する
- 各セクションに本文を入力し、**自社ナレッジDBから過去文章を引用**できる
- これがこのアプリの最大の差別化ポイント

### v2 以降（将来機能）

| バージョン | 機能 |
|-----------|------|
| v2 | 自治体ごとのテンプレート管理 |
| v2 | チームメンバーとの共有・権限管理 |
| v3 | AIによる文章提案（過去データ活用） |
| v3 | 自治体情報の自動取得 |

### 提案書エディタの方針（v1）

- **テキスト管理に徹する（アプローチA）**
- セクション構成は自治体の雛形に従いユーザーが手動登録
- 各セクションにテキストエリアで本文入力
- 自社ナレッジDBからモーダルで引用・挿入できる
- 画像・表・リッチコンテンツはv2以降で対応

### ナレッジ引用の仕組み

```
提案書作成画面のセクション入力中
  → 「ナレッジから引用」ボタンをクリック
  → モーダルが開く（画面遷移なし）
  → カテゴリ・タグ・キーワードで絞り込み検索
  → 「引用する」ボタンでテキストエリアに挿入
  → そのまま編集して仕上げる
```

## Phase 3：DB設計 ✅

### ER図

```
users（ユーザー）
  │
  ├─── projects（案件）
  │         │
  │         └─── proposals（提案書）
  │                   │
  │                   └─── proposal_sections（提案書セクション）
  │
  └─── knowledge_items（自社ナレッジ）
             │
             └─── knowledge_item_tag（中間テーブル）
                        │
                        └─── tags（タグ）
```

### テーブル定義

#### usersテーブル

| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | BIGINT PK | ID |
| name | VARCHAR(100) | 氏名 |
| email | VARCHAR(255) | メールアドレス |
| password | VARCHAR(255) | パスワード（ハッシュ） |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

#### projectsテーブル（案件）

| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | BIGINT PK | ID |
| user_id | BIGINT FK | ユーザーID |
| municipality | VARCHAR(100) | 自治体名 |
| title | VARCHAR(255) | 案件名 |
| deadline | DATE | 締切日 |
| status | ENUM | 準備中/作成中/提出済/採択/不採択 |
| requirements_url | VARCHAR(500) | 募集要項URL |
| template_file | VARCHAR(500) | 雛形ファイルパス |
| memo | TEXT | メモ |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

#### proposalsテーブル（提案書）

| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | BIGINT PK | ID |
| project_id | BIGINT FK | 案件ID |
| title | VARCHAR(255) | 提案書タイトル |
| version | INT | バージョン番号 |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

#### proposal_sectionsテーブル（提案書セクション）

| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | BIGINT PK | ID |
| proposal_id | BIGINT FK | 提案書ID |
| sort_order | INT | 表示順 |
| title | VARCHAR(255) | セクション名（例：会社概要） |
| body | TEXT | 本文 |
| max_pages | TINYINT | ページ数上限・NULL可（自治体指定） |
| note | VARCHAR(255) | 備考・制約メモ・NULL可（例：A4縦・図表含む） |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

#### knowledge_itemsテーブル（自社ナレッジ）

| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | BIGINT PK | ID |
| user_id | BIGINT FK | ユーザーID |
| category | ENUM | 強み/実績/会社概要/その他 |
| title | VARCHAR(255) | タイトル |
| body | TEXT | 本文 |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

#### tagsテーブル

| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | BIGINT PK | ID |
| name | VARCHAR(50) | タグ名 |

#### knowledge_item_tagテーブル（中間テーブル）

| カラム名 | 型 | 説明 |
|---------|-----|------|
| knowledge_item_id | BIGINT FK | ナレッジID |
| tag_id | BIGINT FK | タグID |

> **中間テーブルとは？**
> ナレッジとタグは「多対多」の関係（1つのナレッジに複数タグ、1つのタグが複数ナレッジに使われる）のため、この2つをつなぐ専用テーブルが必要。Laravelでは`belongsToMany`で扱う。

### 主要画面の設計メモ

**ダッシュボード**
- 統計カード（進行中案件数・締切7日以内・ナレッジ登録数・採択数）
- 締切アラート（7日以内の案件を警告表示）
- 案件一覧（締切が近い順にソート）

**提案書作成・編集画面**
- 左カラム：セクション一覧（完了チェック付き）
- 右カラム：テキストエディタ（ページ上限・備考を常時表示）
- 「ナレッジから引用」ボタン → モーダルで検索・引用

**ナレッジ引用モーダル**
- キーワード・カテゴリで絞り込み検索
- 引用ボタンでテキストエリアに直接挿入
- 画面遷移なし（提案書作成を中断しない）
